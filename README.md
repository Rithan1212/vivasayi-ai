# 🌱 VIVASAYI AI — Smart Farmer Advisory System

> A modern, AI-powered web application where farmers can ask farming-related questions in **Tamil or English** and receive instant, expert-style answers — Problem, Reason, Solution, and Precautions — generated by **Google's Gemini AI**.

Built as a college mini-project but engineered to look and feel like a production startup product.

## ✨ Features

- 🤖 **AI Chat** — Ask any crop / pest / soil / weather question and receive a structured advisory
- 🌐 **Bilingual** — Full support for **Tamil (த)** and **English**
- 🔐 **Authentication** — Register, login, logout with validation and JWT (backend) / hashed storage (frontend demo)
- 📊 **Dashboard** — Recent questions, search, view full answers, delete one or clear all history
- 🌙 **Dark / Light mode** — Smooth, system-aware theme toggle
- 📱 **Fully responsive** — Mobile, tablet, desktop
- 🎨 **Premium UI** — Glassmorphism, gradients, framer-motion animations, custom typography
- 🛡️ **Robust error handling** — Friendly messages for offline, timeout, API errors, invalid input
- 🧱 **Clean architecture** — Separation of routes, controllers, services, utilities, and configuration

## 🚀 Installation

### Prerequisites
- **Node.js 18+** and **npm**
- **Python 3.10+** and **pip** (only if you want to run the FastAPI backend)
- A **Gemini API key** — get one free at <https://aistudio.google.com/app/apikey>

### 1. Clone

```bash
git clone https://github.com/your-user/VIVASAYI-ai.git
cd VIVASAYI-ai
```

### 2. Frontend

```bash
npm install
cp .env.example .env
# (optional) edit .env and set VITE_GEMINI_API_KEY for the standalone build
npm run dev
```

Open <http://localhost:5173>.

> The frontend works **out of the box without any API key** — it falls back to a built-in agronomy knowledge base so you can demo and grade without any external dependency. Add `VITE_GEMINI_API_KEY` to enable real Gemini responses.

### 3. Backend (optional but recommended for production)

```bash
cd backend
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# edit .env and paste your GEMINI_API_KEY
uvicorn app.main:app --reload
```

API will be available at <http://localhost:8000>. Interactive Swagger docs at <http://localhost:8000/docs>.

## 🧠 How the AI Works

The Gemini prompt is engineered to always return strict JSON in the following shape:

```json
{
  "problem": "Short one-line description",
  "reason": "2-4 sentence explanation",
  "solution": "Step-by-step actionable plan",
  "precautions": ["bullet 1", "bullet 2", "..."]
}
```

This guarantees consistent, beautifully-rendered advisory cards in the UI. If the model returns malformed JSON or the API fails, the frontend gracefully falls back to a curated knowledge base covering tomato, paddy, aphids, and general agronomy.

## 🔌 API Documentation

All endpoints below are mounted on the FastAPI backend (`http://localhost:8000`). Send `Authorization: Bearer <JWT>` for protected routes.

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `GET`  | `/api/health` | – | Health check |
| `POST` | `/api/auth/register` | – | Create account → returns JWT + user |
| `POST` | `/api/auth/login` | – | Login → returns JWT + user |
| `GET`  | `/api/auth/me` | ✅ | Get current user |
| `POST` | `/api/advisory/ask` | ✅ | Ask the AI: `{ question, language: "en"\|"ta" }` |
| `GET`  | `/api/advisory/history` | ✅ | List the current user's query history |
| `DELETE` | `/api/advisory/history/{id}` | ✅ | Delete a single record |
| `DELETE` | `/api/advisory/history` | ✅ | Clear all history for current user |

Full OpenAPI / Swagger is auto-generated at `/docs`.

## ☁️ Deployment

### Frontend → Vercel

1. Push the repo to GitHub.
2. On Vercel: **New Project** → import the repo.
3. Framework preset: **Vite**. Build command: `npm run build`. Output: `dist`.
4. Add env var `VITE_GEMINI_API_KEY` *only* if you want the standalone client mode. Otherwise leave it blank and point `services/` at your backend URL.
5. Deploy.

### Backend → Render

1. On Render: **New + Web Service** → connect the repo, root = `backend/`.
2. Runtime: **Python 3.11**. Build: `pip install -r requirements.txt`. Start: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`.
3. Add env vars: `GEMINI_API_KEY`, `JWT_SECRET`, `ALLOWED_ORIGINS=https://your-vercel-app.vercel.app`.
4. Deploy.

### Switching the frontend to use the backend

In `src/services/geminiService.ts`, replace the `askGemini` function with a call to `POST {BACKEND}/api/advisory/ask` and add an Axios/fetch wrapper that attaches the JWT from `localStorage`.

## 🔒 Security

- Passwords hashed with **bcrypt** server-side, simple hash client-side demo.
- All inputs validated by **Pydantic** (backend) and explicit checks (frontend).
- JWTs signed with `JWT_SECRET` from `.env` — never committed.
- CORS restricted to whitelisted origins.
- **Gemini API key never reaches the browser** when using the FastAPI backend.
